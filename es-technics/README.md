# Bedrijfs-DNA: ES-Technics BV

De map `es-technics/` bevat de harde standaarden, procedures en templates van het bedrijf. Dit is het fundament voor de consistente werking van de AI-medewerker.

## Inhoud

| Map | Doel |
| --- | --- |
| `procedures/` | Stapsgewijze aanpak voor depannage, onderhoud en inbedrijfstelling. |
| `standaarden/` | Regels omtrent naamgeving, schrijfstijl en veiligheid. Deze worden strikt nageleefd. |
| `templates/` | Sjablonen voor alle gegenereerde documenten. |

## Templates Index

De AI gebruikt ALTIJD deze templates. Afwijken mag niet zonder expliciete toestemming.

*   `as_built_dossier.md` - Oplevering en as-built.
*   `diagnose.md` - Uitgebreid verslag van een storingsdiagnose.
*   `email.md` - Korte, zakelijke e-mails.
*   `installatiefiche.md` - Registratie van een nieuw activum (HVAC/GBS).
*   `klantprofiel.md` - Intake van een nieuwe klant.
*   `offerte.md` - Commercieel voorstel.
*   `opleverdossier.md` - Checklist voor projectoplevering.
*   `parameterlog.md` - Registratie van wijzigingen in GBS of regelaar.
*   `projectfiche.md` - Opstart van een nieuw project.
*   `technisch_rapport.md` - Diepgaande technische analyse.
*   `werkbon.md` - Verslag van een uitgevoerde interventie.

Voor het aanpassen van een template, wijzig het bronbestand in `templates/`. Alle toekomstige documenten volgen dan automatisch de nieuwe opmaak.
